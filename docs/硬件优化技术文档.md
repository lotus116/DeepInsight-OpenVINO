# Intel® DeepInsight 硬件优化技术文档

## 模块概述

本系统的硬件优化分为两个层次：

### 1. 通用硬件优化层 (`hardware/`)
位于 `hardware/` 目录，提供通用的硬件检测、查询缓存、性能遥测功能：
- `__init__.py` (70行) - 模块入口和API导出
- `universal_hardware_optimizer.py` (1100+行) - 核心实现

### 2. 端侧推理优化层 (`rag_engine.py`)
位于 `rag_engine.py`，提供 OpenVINO 专用的 Embedding 模型推理加速：
- OpenVINO 模型转换与优化
- BGE-Small Embedding 模型硬件加速推理
- 向量检索性能优化

## 一、硬件检测功能

### 1.1 支持的硬件平台

| 厂商 | CPU | GPU | 检测方法 |
|------|-----|-----|---------|
| **Intel** | ✅ | ✅ Iris Xe/UHD | wmic (Win) / lspci (Linux) |
| **NVIDIA** | - | ✅ | nvidia-smi |
| **AMD** | ✅ | ✅ Radeon | wmic / lspci |

### 1.2 检测的硬件特性

```python
@dataclass
class HardwareInfo:
    vendor: HardwareVendor      # 主厂商 (Intel/NVIDIA/AMD)
    cpu_model: str              # CPU型号

    cpu_threads: int            # 逻辑线程数
    memory_total: float         # 内存大小 (GB)
    gpu_devices: List[Dict]     # GPU设备列表
    has_avx2: bool              # AVX2指令集
    has_avx512: bool            # AVX512指令集
    has_cuda: bool              # CUDA支持
    has_opencl: bool            # OpenCL支持
```

### 1.3 厂商优先级逻辑

```
优先级: NVIDIA GPU > AMD GPU > Intel CPU > AMD CPU > Unknown
```

如果检测到NVIDIA独立显卡，即使CPU是Intel/AMD，主厂商也会标记为NVIDIA。

## 二、核心优化功能

### 2.1 基于 MD5 与 LRU 的查询缓存机制

**功能**: 缓存相同查询的结果，避免重复计算，提升查询响应速度

| 参数 | 值 |
|------|-----|
| 缓存大小 | 100条 |
| 过期时间 | 5分钟 |
| 淘汰策略 | LRU (最近最少使用) |
| 键生成 | MD5 哈希 |

**实现逻辑**:
```python
def get_cached_result(cache_key):
    if key in cache and not expired:
        return cache[key]  # 命中
    return None  # 未命中

def set_cached_result(cache_key, result):
    if len(cache) >= 100:
        delete_oldest()
    cache[key] = {result, timestamp}
```

**使用方式**:
```python
from hardware import get_universal_optimizer
opt = get_universal_optimizer()

key = opt.generate_cache_key("SELECT * FROM users", param1)
cached = opt.get_cached_result(key)
if cached is None:
    result = execute_query()
    opt.set_cached_result(key, result)
```

### 2.4 计时执行 (Timed Execution)

**功能**: 自动记录函数执行时间，用于性能分析

**返回**: `(结果, 执行时间毫秒)`

**实现逻辑**:
```python
def timed_execution(func, *args):
    start = time.perf_counter()
    result = func(*args)
    end = time.perf_counter()
    
    execution_ms = (end - start) * 1000
    self.execution_times.append(execution_ms)
    
    return result, execution_ms
```

**使用方式**:
```python
result, time_ms = opt.timed_execution(my_function, arg1, arg2)
print(f"执行时间: {time_ms:.2f}ms")
```

## 三、查询分析功能

### 3.1 查询复杂度评分

系统分析SQL查询并计算0-1的复杂度分数：

| 特征 | 加成 |
|------|------|
| 基础复杂度 | +0.1 |
| JOIN操作 | +0.3 基础, +0.1/每个JOIN |
| 聚合函数(SUM/COUNT等) | +0.2 |
| 子查询 | +0.25 |
| 窗口函数 | +0.2 |
| ORDER BY | +0.1 |
| LIMIT/TOP | -0.05 |

### 3.2 动态因子

性能估算考虑以下动态因子：

| 因子 | 范围 | 说明 |
|------|------|------|
| 复杂度因子 | 0.8-1.3 | 查询越复杂，优化收益越大 |
| 数据量因子 | 0.9-1.4 | 小数据集优化效果更好 |
| 查询类型因子 | 0.85-1.2 | SELECT最优，DELETE最差 |
| 学习因子 | 1.0-1.15 | 使用次数越多效果越好 |

## 四、性能估算公式

### CPU优化估算
```
base = 0.1  # 基础10%
+ 0.15 (if cores >= 4)
+ 0.10 (if cores >= 8)
+ 0.20 (if AVX2)
+ 0.15 (if AVX512)
+ 0.10 (if Intel)

CPU提升 = base × 动态因子
上限: 85%
```

### GPU加速估算
```
base = 1.0
+ 1.5 (if NVIDIA + CUDA)
+ 0.3 (if Intel GPU)
+ 1.2 (if AMD GPU)
+ 0.2 (if OpenCL)

GPU加速比 = base × 动态因子
上限: 3.5x
```

### 总体加速比
```
overall = (CPU因子 × 0.4) + (GPU因子 × 0.4) + (内存效率 × 0.2)
上限: 3.50x
```

## 五、端侧推理优化（rag_engine.py）

> **注意**: 本节描述的优化功能位于 `rag_engine.py` 中的 `IntelRAG` 类，而非 `hardware/` 模块。

### 5.1 OpenVINO 模型优化

**功能**: 使用 Intel OpenVINO Runtime 对 BGE-Small Embedding 模型进行硬件加速推理

**优化技术**:
- **算子融合**: 合并多个算子以减少计算开销
- **内存优化**: 优化内存布局，减少数据拷贝
- **指令集加速**: 利用 AVX-512、VNNI 等指令集
- **图优化**: 优化计算图结构，消除冗余计算

**性能指标**:
| 指标 | PyTorch FP32 | OpenVINO FP32 | 加速比 |
|------|--------------|---------------|--------|
| 平均延迟 | 3.4±0.1ms | **1.0±0.1ms** | **3.50x** |
| P95 延迟 | 4.4ms | 1.6ms | 2.75x |
| 吞吐量 | 292.8 QPS | 1013.8 QPS | 3.50x |

### 5.2 架构位置

```
rag_engine.py
├── IntelRAG (类)
│   ├── __init__(): 初始化 OpenVINO Runtime
│   ├── _load_openvino_model(): 加载 OpenVINO 优化模型
│   ├── embed_with_openvino(): OpenVINO 推理接口
│   └── embed_with_pytorch(): PyTorch 回退接口
└── 模型文件
    └── models/bge-small-ov/ (OpenVINO IR 格式)
        ├── openvino_model.xml
        └── openvino_model.bin
```

### 5.3 与 hardware/ 模块的关系

| 层次 | 模块 | 功能 | 加速目标 |
|------|------|------|----------|
| **通用硬件优化** | `hardware/` | 查询缓存、性能遥测 | SQL查询、数据处理 |
| **端侧推理优化** | `rag_engine.py` | OpenVINO Embedding 推理加速 | 向量检索、语义匹配 |

**协同工作**:
1. `rag_engine.py` 使用 OpenVINO 进行 Embedding 向量计算（3.50x 加速）
2. `hardware/` 模块提供查询缓存，避免重复向量计算
3. `hardware/` 模块提供线程数建议与性能状态面板，为前端展示与调度提供参考
