核心控制流

  1. 用户输入：通过 Streamlit 界面输入自然语言查询
  2. 预处理：查询预处理、Prompt 模板加载、上下文记忆检索
  3. RAG 检索：调用 OpenVINO 优化的 bge-small-zh-v1.5 模型执行二阶段混合检索
  4. SQL 生成：常规使用 DeepSeek-Chat，重试阶段按配置可切换至 DeepSeek-Reasoner
  5. 执行与优化：查询执行、性能遥测与状态展示、结果处理、Plotly 可视化
  6. 错误处理：执行失败时触发 Agent 错误感知重试机制
  7. 输出：返回结果、生成洞察、支持导出
